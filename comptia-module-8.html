Module 8: Infrastructure Threats and Security Monitoring

Attacks on networks, security monitoring, and email defenses.

I. Infrastructure Threats: Attacks on Networks

Network attacks come in various forms, targeting different layers of communication or specific protocols.

A. On-Path Attacks

An on-path attack occurs when a threat actor places themselves in the middle of communication between two users or devices. A significant advantage of this type of attack is that it can proceed without the two targets being aware of the attacker’s presence. The attacker gains flexibility, allowing them to either eavesdrop to gather valuable information or modify the message before sending it to the recipient.

1. Man-in-the-Middle (MITM) Attacks

In a Man-in-the-Middle (MITM) attack, a threat actor positions themselves within a communication path between two parties. The goal of an MITM attack is typically to eavesdrop on the conversation or impersonate one of the legitimate parties. A typical MITM attack involves two phases: first, intercepting the traffic, and second, decrypting the transmissions.

2. Replay Attacks

A replay attack involves making a copy of a legitimate transmission and then using that copy at a later time. A specific type, the session replay attack, intercepts and subsequently uses a session ID to impersonate a user. Threat actors utilize several techniques to steal an active session ID, including network attacks (like MITM impersonation) and endpoint attacks (such as cross-site scripting, Trojans, and malicious JavaScript coding).

3. Man-in-the-Browser (MITB) Attacks

A man-in-the-browser (MITB) attack intercepts communication, but specifically between a browser and the underlying computer, often to steal or manipulate data. This type of attack usually starts with a Trojan infecting the computer and installing an “extension” into the browser configuration. When the browser is launched, the extension is activated and waits for a specific webpage (such as a financial institution site) where a user enters sensitive information. When the user clicks “Submit,” the extension captures all the data from the form fields.

B. Domain Name System (DNS) Attacks

The Domain Name System (DNS) is crucial as a hierarchical name system responsible for matching computer names to corresponding IP addresses. A DNS-based attack replaces a legitimate DNS address, silently redirecting the user's computer to a different device.

A successful DNS attack has two major consequences:

  1. URL redirection.
  2. Domain reputation damage.
DNS Attack Type Description Target
DNS Poisoning Modifies a local host file on a device to point to a different domain, or adds an entry directing the computer to an attacker-controlled DNS server. Local device or the local DNS cache
DNS Hijacking Infects an external DNS server with IP addresses that point to malicious sites. This has the advantage of redirecting all users accessing that server. Attackers attempt to exploit a protocol flaw to convince the authentic DNS server to accept fraudulent entries. External DNS Server

C. Distributed Denial of Service (DDoS) Attacks

A denial of service (DoS) attack is a deliberate attempt to prevent authorized users from accessing a system by overwhelming it with requests. Most DoS attacks today are distributed denial of service (DDoS) attacks, which utilize hundreds or thousands of devices to flood the target server with requests. The endpoints participating in a DDoS attack are typically infected and controlled by threat actors, meaning the legitimate users of those devices are completely unaware that their computers are part of the attack.

Attackers use two main means to generate data sufficient to overwhelm a system:

  1. Using a large number of compromised devices, with each device sending bogus requests.
  2. Using amplified attacks (reflection attacks), where threat actors target a misconfigured Internet device or service. This causes the misconfigured device/service to reflect and generate an even larger payload directed at the ultimate target.

D. Malicious Coding and Scripting Attacks

Attacks on networks are frequently facilitated by malicious software code and scripts.

Scripting/Coding Environment Description and Attack Context
PowerShell A task automation and configuration management framework from Microsoft. Tasks are performed by specialized .NET classes called cmdlets. Attackers can inject code from the PowerShell environment into other processes without first storing any malicious code on the hard disk.
Visual Basic for Applications (VBA) An event-driven Microsoft programming language used to automate complex or repeated tasks. VBA is often used to create macros. Though dating back to the late 1990s, macros continue to be a key attack vector.
Python A programming language runnable on several OS platforms. Best practices for security include using the latest version, staying current on vulnerabilities, being careful when formatting strings, and downloading only vetted libraries.
Bash The command language interpreter for the Linux/UNIX OS. Attackers utilize Bash scripting. Exploits have taken advantage of vulnerabilities, for example, one vulnerability allowed attackers to remotely attach a malicious executable file to a variable that is executed when Bash is invoked.

E. Layer 2 Attacks

The OSI reference model separates networking steps into seven layers, each performing specific tasks. Layer 2, the Data Link Layer, is responsible for dividing data into packets. A compromise at Layer 2 can potentially affect the entire communication.

1. Address Resolution Protocol (ARP) Poisoning

If the IP address for a device is known but the MAC address is not, the sending computer uses the Address Resolution Protocol (ARP) packet to determine the corresponding MAC address. These MAC addresses are stored in an ARP cache for future reference. ARP poisoning relies upon MAC spoofing, which is the imitation of another computer achieved by changing the MAC address.

2. MAC Spoofing Attacks

Two common attacks involving the spoofing of MAC addresses are MAC cloning and MAC flooding.

  • MAC Cloning Attack: Threat actors discover a valid MAC address of a device connected to a switch and then spoof that MAC address on their own device. Consequently, the switch changes its MAC address table to link the cloned MAC address with the port to which the attacker’s device is connected.
  • MAC Flooding Attack: A threat actor overflows the switch with spoofed Ethernet packets, with every packet containing a different source MAC address. This causes the switch to enter a failure mode, broadcasting all packets to all devices.

F. Credential Relay Attacks

A credential relay attack attempts to steal authentication credentials and then use them to access a system. Specifically, attackers intercept digests of user passwords as they are being transmitted and then relay the clients’ credentials to impersonate the legitimate user.

II. Security Monitoring and Alerting

Monitoring involves examining network traffic, activity, transactions, or behavior to detect security-related anomalies.

A. Monitoring Methodologies

There are four primary methodologies used for monitoring and threat detection:

Monitoring Methodology Definition and Mechanism Key Concept
Anomaly Monitoring Designed for detecting statistical anomalies. A secure baseline of normal activities is compiled, and an alarm is raised if there is a significant deviation from this reference set of data. Deviation from a Baseline
Signature-based Monitoring Examines network traffic, activity, transactions, or behavior to look for well-known patterns. These activities are compared against a predefined signature. Matching Predefined Patterns
Behavior-based Monitoring Uses the “normal” processes and actions as the standard. It continuously analyzes the behavior of processes and programs on a system and alerts the user if any abnormal actions are detected. Detecting Abnormal Actions
Heuristic Monitoring Founded on experience-based techniques. It attempts to answer the question, “Will this do something harmful if it is allowed to execute?”. Experience-based Evaluation

B. Monitoring Activities

Monitoring is composed of several continuous and discrete activities:

  • Scanning: A frequent and ongoing process, often automated, that continuously searches for evidence of an attack.
  • Reporting: Generating documentation on the results of monitoring activities.
  • Quarantine: Isolating systems that have been compromised.
  • Alerting: Detecting and notifying operators about meaningful events that may denote an attack.
  • Alert tuning: “Tweaking” the alerting function to weed out false positives.
  • Archiving: Retaining historical documents and records of monitoring.

C. Tools for Monitoring and Alerting

Organizations use specialized tools for collecting, analyzing, and acting upon security data.

1. Packet and Flow Analysis

Collecting and analyzing data packets that cross a network provides valuable security information, often examining the entire contents of the packet.

  • Packet Analysis Tools:
    • Wireshark: A popular GUI packet capture and analysis tool.
    • Tcpdump: A command-line packet analyzer.
    • Tcpreplay: A tool for editing packets and then replaying them back onto the network to observe their behavior.
  • Flow Analysis (Network Traffic Analysis): This is the process of monitoring network devices and sounding an alert if traffic exceeds a predetermined baseline. Flow analysis for information security differs from traditional network traffic flow analysis because it eliminates monitoring agents, uses deep packet inspection, and provides richer information. NetFlow is a session sampling protocol that specifically collects IP network traffic.

2. Data Loss Prevention (DLP)

Data loss prevention (DLP) is a system of security tools used to recognize, identify, and ensure protection for data that is critical to the organization. Most DLP systems rely on content inspection, which is a security analysis of the transaction within its approved context. Administrators create DLP rules, which are uploaded to a DLP server. If a policy violation is detected by a DLP agent, it is reported back to the DLP server.

3. General Monitoring Tools

  • Simple Network Management Protocol (SNMP): A protocol used to remotely monitor, manage, and configure devices on the network. An SNMP trap is a specific type of Protocol Data Unit (PDU) that sends an unsolicited message to the manager regarding critical events occurring in the managed device.
  • Log Aggregation: This practice enables security personnel to gather events from disparate sources into a single entity, making it searchable and analyzable.
  • Security Content Automation Protocols (SCAP): These protocols help automate vulnerability management and determine whether the enterprise is compliant with required security policies.

4. SIEM and SOAR Solutions

Two advanced systems consolidate and automate security operations:

Tool Description Key Features
Security Information and Event Management (SIEM) Consolidates real-time security monitoring and the management of security information with the analysis and reporting of security events. Aggregation, Correlation, Automated alerting and triggers, Time synchronization, Event duplication and logs.
Security Orchestration, Automation, and Response (SOAR) Similar to a SIEM, designed to help security teams manage and respond to a high volume of warnings and alarms. However, SOAR takes it further by combining comprehensive data gathering and analytics to automate incident response. Comprehensive data gathering, Analytics, Automated incident response.

III. Email Defenses

Email security is vital given the common use of email as an attack vector.

A. Email Components and Threats

The basic components for sending and receiving email include the Mail User Agent (MUA) and the Mail Transfer Agent (MTA). The MUA is the program used to read and send email from an endpoint. MTAs are programs that accept email messages from senders and route them toward their recipients. As email is transferred from one MTA to the next, information, including an analysis of the email by the MTA, is added to the email header.

Threats related to email include:

  • Malicious payload
  • Embedded links
  • Impersonation
  • Forwarding

General defenses like spam and phishing filters can address phishing attacks, and anti-malware can minimize malicious payloads. Specialized protocols and gateways provide more robust protection.

B. Email Authentication and Security Mechanisms

Defense Mechanism Function and Mechanism Key Purpose
Sender Policy Framework (SPF) An email authentication method that identifies the MTA email servers authorized to send email for a domain. Protects a domain from spoofing and helps prevent valid domain messages from being marked as spam.
Domain Keys Identified Mail (DKIM) An authentication technique that validates the content of the email message itself. This validation is accomplished through a digital signature. Validates message content integrity.
Domain-based Message Authentication, Reporting, and Conformance (DMARC) Allows the administrative owner of a domain to publish a policy in their DNS records. This policy specifies which mechanism (DKIM, SPF, or both) must be used when sending email from that domain. Enforces SPF and/or DKIM policies and enables reporting.
Secure Email Gateway (SEG) Acts as a “proxy” for the organization’s email server. The SEG can filter and inspect emails for malicious content. Filters and inspects malicious content before reaching the internal server.

A Secure Email Gateway (SEG) has limitations, including potentially providing only single-layer security, exposing protections, and issues with multiple root domains.

Study Guide for Module 8

Quiz: Short-Answer Questions

1. What is the primary goal of a Man-in-the-Middle (MITM) attack?

The primary goal is to eavesdrop on the conversation or impersonate one of the legitimate parties to steal information or manipulate data.

2. Explain the difference between a Replay attack and a Session Replay attack.

A standard Replay attack copies a legitimate transmission to use later. A Session Replay attack specifically intercepts and uses a valid session ID to impersonate a user and hijack their active session.

3. How does a DNS Poisoning attack differ from a DNS Hijacking attack?

DNS Poisoning targets the local device's host file or DNS cache. DNS Hijacking infects an external DNS server to redirect all users accessing that server to malicious sites.

4. What is the main characteristic of a Distributed Denial of Service (DDoS) attack?

It utilizes hundreds or thousands of compromised devices (botnet) to flood a target server with requests, overwhelming it and denying access to legitimate users.

5. Describe the "Amplified" or "Reflection" technique used in DDoS attacks.

Attackers send requests to a misconfigured device/service spoofing the target's IP. The device responds (reflects) with a much larger payload (amplification) directed at the target.

6. What is the purpose of Address Resolution Protocol (ARP) Poisoning?

To associate the attacker's MAC address with the IP address of a legitimate device (like the default gateway) in the victim's ARP cache, allowing the attacker to intercept traffic.

7. Define "MAC Spoofing" and name two common attacks that utilize it.

MAC Spoofing is imitating another computer by changing the MAC address. Two common attacks are MAC Cloning (copying a valid MAC) and MAC Flooding (overflowing the switch table).

8. What is the difference between Anomaly Monitoring and Signature-based Monitoring?

Anomaly Monitoring detects deviations from a baseline of normal activity. Signature-based Monitoring looks for known patterns or "signatures" of specific attacks.

9. Explain the function of a Security Information and Event Management (SIEM) system.

A SIEM consolidates real-time security monitoring and log management from disparate sources, providing aggregation, correlation, and automated alerting for security events.

10. What is the purpose of the Sender Policy Framework (SPF) in email security?

SPF is an email authentication method that identifies which mail servers are authorized to send email on behalf of a domain, preventing spoofing.

Essay Questions

  1. Analyze the impact of Infrastructure Threats on modern network security. Discuss how attacks like MITM, DDoS, and DNS poisoning disrupt operations and what comprehensive strategies organizations should employ to mitigate these risks.
  2. Evaluate the effectiveness of different Security Monitoring Methodologies. Compare Anomaly, Signature, Behavior, and Heuristic monitoring. Which methodology (or combination) provides the most robust defense against zero-day threats vs. known vulnerabilities?
  3. Discuss the role of SIEM and SOAR in a Security Operations Center (SOC). How do these tools complement each other? Explain how automation in SOAR can reduce the "mean time to respond" (MTTR) to security incidents.
  4. Examine the vulnerabilities associated with Email as an attack vector. Why is email such a prevalent target? Discuss the importance of implementing SPF, DKIM, and DMARC in concert to secure organizational communication.
  5. Explain the significance of Layer 2 attacks and their mitigation. Why are attacks like ARP Poisoning and MAC Spoofing dangerous even if higher-layer security is in place? What specific controls can be implemented at the switch level to prevent them?

Glossary of Key Terms

Term Definition
ARP Poisoning An attack that corrupts the ARP cache, associating the attacker's MAC address with a legitimate IP.
DDoS (Distributed Denial of Service) An attack using multiple computers (botnet) to overwhelm a target with traffic.
DKIM (Domain Keys Identified Mail) Email authentication method that validates a message's domain name identity through cryptographic authentication.
DMARC An email authentication protocol that gives domain owners the ability to protect their domain from unauthorized use (spoofing).
DNS Hijacking An attack that infects an external DNS server with malicious IP addresses.
DNS Poisoning An attack that modifies a local host file or DNS cache to redirect traffic.
MITM (Man-in-the-Middle) An attack where the attacker secretly relays and possibly alters the communications between two parties.
SIEM (Security Information and Event Management) A solution that provides real-time analysis of security alerts generated by applications and network hardware.
SOAR (Security Orchestration, Automation, and Response) A solution stack of compatible software programs that allow an organization to collect data about security threats and respond to low-level security events without human assistance.
SPF (Sender Policy Framework) An email authentication method that specifies the mail servers authorized to send email on behalf of your domain.
