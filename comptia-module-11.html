I. Introduction to Cloud Computing

What is Cloud Computing?

Cloud computing is a model for enabling convenient, on-demand network access to a shared pool of configurable computing resources (such as networks, servers, storage, applications, and services) that can be rapidly provisioned and released with minimal management effort or service provider interaction. Entities that offer cloud computing are known as cloud service providers (CSPs).

Historically, enterprises employed a centralized architectural model where servers and networking equipment were installed locally, requiring the company to purchase, install, and manage all necessary hardware and software, leading to spiraling costs and difficulty reaching optimal IT levels due to supply chain issues or personnel shortages. Cloud computing shifts costs from capital expenditures (CapEx)—purchasing and installing infrastructure—to operating expenses (OpEx), where costs are based solely on usage.

Cloud computing offers a much more flexible approach compared to traditional hosting services, supporting multiple tenants online while providing rapid and automatic scalability and elasticity. Because cloud resources are available online, users from virtually anywhere can access them using a web browser without needing additional software installed.

Key Characteristics and Cost Savings of Cloud Computing

The flexibility and cost savings inherent in cloud computing are due to several characteristics:

Characteristic Explanation
Elasticity and Scalability Resources can be quickly expanded or reduced to meet specific service requirements. Users can provision resources for a specific task (e.g., peak holiday e-commerce traffic) and then drop those resources when no longer needed.
Pay-per-use (Metered services) Fees are based on the computing resources consumed, whether for a short term (e.g., one day) or a longer duration.
On-demand self-service The consumer can make changes, such as increasing or decreasing computing resources, without requiring human interaction from the service provider. Services are only activated when needed.
Resiliency Cloud services can completely isolate the failure of a server or storage resource from users, as the provider migrates the service to a different resource without the user's knowledge. This eliminates the need for organizations to maintain idle excess capacity.
Universal client support Virtually any networked device (desktop, laptop, smartphone, tablet, etc.) can access cloud computing resources.
Invisible resource pooling Physical and virtual resources are pooled to serve multiple consumers simultaneously, dynamically assigned based on needs; the customer has little or no control or knowledge of the physical location of the resources.
Immediate elasticity Computing resources can be increased or decreased quickly to meet demands.

Regulatory and Market Concerns

Despite the popularity of cloud computing, regulatory bodies have raised concerns about the heavy reliance on a small number of providers.

  • In early 2023, the U.S. Treasury Department warned that financial institutions relying on few CSPs risks could lead to widespread disruption if one provider experienced a technical issue or cyberattack, immediately impacting consumers.
  • The UK’s communications regulating body, Ofcom, expressed concern about the practices of AWS and Microsoft, which control 60 to 70 percent of the UK cloud market. This concentration could lead to customers paying more or smaller providers being squeezed out.
  • The European Union (EU) passed the Digital Operational Resilience Act (DORA) to ensure financial system participants have safeguards to mitigate risks, introducing an oversight framework for critical third-party providers like CSPs.

II. Cloud Infrastructure: Types, Locations, and Architecture

Types of Clouds

Clouds are categorized based on their accessibility and deployment model:

Cloud Type Description Security/Control Implications
Public Cloud Services and infrastructure are offered to all users with access provided remotely through the Internet. Open to anyone.
Community Cloud Open only to specific organizations that have common concerns, such as organizations needing to adhere to strict regulatory requirements like HIPAA. Shared security model among specific organizations.
Private Cloud Created and maintained on a private network. Offers the highest level of security and control, but reduces cost savings as the company must purchase and maintain all hardware and software.
Hybrid Cloud A combination of public and private clouds. Offers flexibility but introduces special security challenges and often requires shared security responsibility.

Cloud Locations

Cloud computing operates using a decentralized model rather than the traditional centralized model. Computing resources may be located in several distinct areas:

Location Description Example
On-premises (On-prem) Computing resources located on the campus of the organization. Desktop computer, local area network, data center.
Off-premises A computing resource hosted and supported by a third party. Remote backup facility.
Fog A decentralized computing infrastructure where data, compute capabilities, storage, and applications are located between the data source and the cloud. Automated guided vehicles on an industrial shop floor.
Edge Computing that is performed at or very near to the source of data instead of relying on the cloud or on-prem for processing. Internet of Things (IoT) device.
Cloud A remote facility for computing. Artificial intelligence processing engine.

Cloud Architecture Components

Several elements define cloud architecture:

  • Thin Client: A computer that runs from resources stored on a central cloud server instead of a localized hard drive. Thin clients connect remotely where applications and data are stored and processing takes place.
  • Transit Gateway: An Amazon Web Services (AWS) technology that allows organizations to connect all existing virtual private clouds (VPCs), physical data centers, remote offices, and remote gateways into a single managed source. It offers a "hub-and-spoke" network topology allowing full control and monitoring over resources.
  • Serverless Infrastructure: An architecture where capacity planning, installation, setup, and management are invisible to the user because they are handled by the cloud provider. Although servers are performing the critical functions, the end-user does not manage them.

III. Cloud Service Models

Cloud computing offers various service models, defining the responsibility split between the provider and the consumer:

Cloud Model Description User Responsibility Level Example Scenario
Software as a Service (SaaS) The vendor provides access to their software applications running on a cloud infrastructure. It eliminates the burden of purchasing, installing, maintaining, and upgrading software locally. Minimal. Software is offered as a complete service. Using a customer relationship management (CRM) application accessed via a web browser.
Platform as a Service (PaaS) Consumers install and run their own specialized applications on the cloud computing network. Provides a software platform or development framework to build, debug, and deploy apps. Moderate. Control over applications, but cloud provider guarantees elasticity and scalability. An organization developing and deploying a new company application (Arsene's Project).
Infrastructure as a Service (IaaS) The vendor allows customers to deploy and run their own software, including Operating Systems (OSs) and applications, using unlimited "raw" computing, storage, and network resources. High. Enterprise is responsible for monitoring and requesting additional services (scaling and elasticity are not always automatic). Building a completely customized virtual infrastructure, including selecting CPU speed, memory, and networking resources.
Anything as a Service (XaaS) A broad category of subscription services related to cloud computing. XaaS is any IT function or digital component transformable into a service. Varies widely based on the service offered. Security as a Service (SECaaS), which provides security services like intrusion detection and SIEM from the cloud.

IaaS is often discussed in the context of network architecture, such as the traditional three-tier architecture (Web servers, Application servers, Database servers). An enterprise must decide whether to migrate all three tiers to the cloud or keep sensitive components, like Tier 3—Database servers, on-prem for security.

IV. Cloud Management and Application Design

Cloud Management

Ongoing management of cloud resources can be complex, especially if an organization contracts with multiple CSPs. Management can be handled either locally by the organization or by contracting with a third-party management service provider.

Local Management:
One challenge in local cloud management is services integration, which attempts to achieve a "boundary-less" approach by integrating all users and services (SaaS, PaaS, on-prem apps, etc.) across the enterprise. Locally managing the cloud requires written resource policies that clearly outline who is responsible, their duties, how the cloud can be used, and the acquisition process. A significant security threat associated with local management is "shadow IT" cloud environments, often set up by employees using personal credit cards without authorization.

Service Providers:
Many organizations use external third-party service providers:

  • Managed Service Provider (MSP): Delivers services (network, application, infrastructure, security) through ongoing, regular support and active administration. An MSP essentially assumes the role of a traditional on-prem IT organization and can manage resources on the customer’s premises, in their own data center, in a third-party data center, or in a cloud environment.
  • Managed Security Service Provider (MSSP): A specialized type of MSP that assists with or fully assumes cybersecurity defenses, including installing and monitoring firewalls, VPNs, antivirus, and intrusion detection systems.

Cloud-Native Microservices

Traditional application design is often called "monolithic" because the entire program is developed as a single entity. As monolithic applications grow, deployment times lengthen, the codebase becomes difficult for single teams to manage, and modifications often destabilize other parts of the code.

The solution is to divide the program into smaller, specialized parts of code known as microservices (or cloud-native microservices).

  • Microservices Architecture: This architecture utilizes smaller, more specialized elements. Each element manages its own database, generates its own logs, and handles user authentication, often performed using specialized microservices APIs (like RESTful APIs).
  • Advantages: Microservices allow for easier code updates, permit programmer teams to use different programming languages for different components, and enable components to be scaled up or down independently, reducing waste and cost.

V. Cloud Security: Defenses, Vulnerabilities, and Controls

Cloud-Based Security Mechanisms

Virtual security devices are essential for providing protection in a cloud computing environment.

Security Device/Service Description
Cloud Firewall Virtual software that functions similarly to a physical security appliance by examining traffic entering and leaving the cloud. Can scale quickly and is deployed in the public cloud.
Secure Web Gateway (SWG) A product that examines both incoming and outgoing traffic, performs basic URL and web application monitoring, analyzes encrypted traffic (SSL), performs Data Loss Prevention (DLP), and provides alerts to monitoring devices like SIEM appliances. Can be placed on endpoints, at the edge, or in the cloud.
Cloud Access Security Broker (CASB) A set of software tools or services that resides between an enterprise’s on-prem infrastructure and the cloud provider’s infrastructure. It acts as a "gatekeeper," ensuring that the enterprise's security policies (like data encryption) extend to its data stored in the cloud.
Secure Access Service Edge (SASE) A model representing the convergence of several security services into a single, cloud-delivered service. SASE identifies users and devices, applies policy-based security based on identity and real-time context, and provides secure access to applications or data. Technologies making up SASE include SWG, CASB, Zero-Trust Architecture (ZTA), and Wide Area Network (WAN) technologies.
Security Services Edge (SSE) A subset of SASE that includes only the security components (SWG, CASB, ZTA) and excludes WAN technologies.

Cloud Vulnerabilities

While the cloud offers enhanced security options, it also presents unique vulnerabilities (called cloud-specific vulnerabilities).

Security Issue Description
Unauthorized access to data Can result from improper cloud security configurations leaving data exposed.
Lack of visibility Organizations often have limited or no visibility into the security mechanisms of the cloud provider, making it difficult to verify the effectiveness of security controls.
Insecure APIs Threat actors can exploit a vulnerable API, even though APIs help cloud customers customize PaaS by providing data recognition, access, and effective encryption.
Compliance regulations Difficult to maintain compliance because it requires knowing the location, access permissions, and protection status of data, which is challenging in an opaque cloud system.
System vulnerabilities The cloud infrastructure is prone to system vulnerabilities due to complex networks and multiple third-party platforms.

Two primary challenges associated with cloud use are conceptual confusion:

  1. Responsibility Matrix Confusion: Determining who is responsible for which security element is challenging. A responsibility matrix lists various duties, noting that some are always the organization's duty, some belong to the CSP, and others may be shared. For example, based on a common matrix, the cloud provider is responsible for Physical hosts, Physical datacenters, and Physical networks, but not for Devices used by the customer. Confusion often leads to security being incorrectly applied.
  2. Lack of Cloud Conceptual Model: Physical networks map neatly to the Open Source Interconnection (OSI) seven-layer model, which standardizes network functionality. In cloud computing, the OSI model is less useful because the provider manages physical security and connectivity, and everything the customer sees is abstract and virtual (existing as code). The lack of a conceptual model makes selecting and managing virtual security devices more challenging. Although different cloud-based conceptual models are being proposed, no single standard has been widely adopted.

Cloud Security Controls

Security controls, which can be policies, procedures, techniques, or devices, mitigate risk. These controls can be cloud native controls (inherent to the platform) or available from third-party vendors.

Control Measure Description
Conducting Audits A cloud security audit is an independent examination of cloud service controls to render an objective assessment of security, availability, and performance. Audits verify conformance to established standards for compliance.
Utilize Regions and Zones Reliability and resiliency are achieved by duplicating processes across one or more geographical areas. This is called high availability across zones. AWS maintains multiple geographic Regions, each containing Availability Zones (AZs)—data centers with redundant power, networking, and connectivity, physically separated to protect against localized disasters.
Secrets Management Enables strong security and improved management of a microservices-based architecture. It provides a central repository and single source to manage, access, and audit "secrets" (like API keys, passwords, certificates, encryption keys, and tokens) needed for microservices to communicate, preventing them from being hard-coded into software.
Enforce Functional Area Mitigations Security mitigations must be applied across the three functional areas of cloud computing: storage, network, and compute.

Hybrid Cloud Security Considerations

Hybrid clouds pose hybrid cloud considerations—special security challenges stemming from spanning both public and private spaces, often resulting in shared security responsibilities.

Hybrid Cloud Security Best Practices include:

  • Encrypting sensitive data and data traffic between the two clouds and inspecting all encrypted traffic.
  • Monitoring and auditing configurations and using automation, rather than manual management, to keep configurations aligned.
  • Running regular scans to identify weak points.
  • Securing all endpoints.
  • Enforcing zero-trust security.

VI. Virtualization Technology

Defining Virtualization

Virtualization is a means of managing and presenting computer resources by function without regard to their physical layout or location. It allows the consolidation of multiple physical servers into Virtual Machines (VMs) that run on a single physical computer. The widespread adoption of server virtualization is directly linked to the popularity of cloud computing.

  • Host Virtualization: A type of virtualization in which an entire operating system environment is simulated.
  • Virtual Machine (VM): A simulated software-based emulation of a computer. The physical OS is the "host system," and the foreign virtual OS is the "guest system".
  • Hypervisor: The VM monitor program that manages the VM operating systems, allocating resources like memory and I/O functions in real time.

Types of Hypervisors

Hypervisor Type Description Alternate Name(s)
Type I Runs directly on the computer’s hardware instead of the underlying operating system. "Native" or "Bare Metal" hypervisors.
Type II Runs on the host OS, much like a regular application. N/A

Containerization and VDI

A reduced instance of virtualization is containerization.

  • Container: Holds only the necessary OS components (binary files and libraries) required for a specific application to run. Containers reduce necessary storage and RAM, start more quickly than full VMs because the entire OS doesn't need to boot, and can easily be moved between computers.
  • Virtual Desktop Infrastructure (VDI): A common application of virtualization where a user desktop runs inside a remote VM located on a server. VDI enables personalized desktops accessible from any device that can access the server, supporting mobile users and allowing centralized management for troubleshooting.

Advantages of Virtualization

Virtualization provides numerous benefits, contributing to host availability and elasticity:

  1. Cost Reduction: Fewer physical computers need to be purchased and maintained, reducing hardware, maintenance, and electricity/cooling costs.
  2. Host Availability/Elasticity: New virtual server machines can be quickly made available, and resources (like RAM or storage) can be easily expanded or contracted as needed.
  3. Uninterrupted Access (Live Migration): Virtualization supports live migration, allowing a VM to be moved to a different physical computer (by storing its current state on shared storage) with no impact or noticeable interruption to users. This is also useful for load balancing.

VII. Infrastructure as Code (Virtualization Applications)

Instances of virtualization that apply to networking components are referred to as infrastructure as code.

Software-Defined Network (SDN)

A Software-Defined Network (SDN) virtualizes parts of the physical network so that it can be reconfigured more quickly and easily. It solves the bottleneck where virtual components (VMs and VLANs) conflict with static physical network hardware (switches and routers).

SDN achieves this flexibility by separating the control plane from the data plane.

  • Control Plane: Consists of one or more SDN controllers that execute complex functions like routing and security checks, defining the data flows.
  • Data Plane: The physical network switches manage simple "flow tables" whose entries are created by the controller.

When traffic needs to flow, it receives permission from the SDN controller, which verifies communication against enterprise policy, computes a route, and adds entries to the switches. This architecture simplifies the network for applications and allows for rapid creation and deployment of applications to orchestrate traffic flow based on performance or security requirements. From a security perspective, SDN can simplify VLAN extension to secure data and ensure all network traffic is routed through a firewall.

Other Infrastructure as Code Examples

  • Software-Defined Wide Area Network (SD-WAN): A virtualized service that connects and extends enterprise WAN networks over large geographical distances. SD-WAN addresses challenges associated with traditional WANs and can adapt quickly to changing conditions, offering better security and reliability than traditional WANs because it is based on software, not hardware.
  • Software-Defined Visibility (SDV): A framework that allows users to create programs to automate critical security functions that previously required manual intervention. SDV enables automation of dynamic response to threat patterns, adjustments to traffic mode configurations for in-line security tools, and additional IT operations-management functions.

VIII. Virtualization Security Concerns and Advantages

Security Advantages of Virtualization

Virtualization offers numerous security-related advantages:

Security Feature Description
Testing Security Updates Latest security updates can be downloaded and run on a VM before installation on production computers to check compatibility and impact.
Snapshots A snapshot of a VM’s state can be saved and later reloaded to revert the VM to a prior state if modifications cause issues.
Security Control Testing Existing security configurations can be tested (security control testing) using a simulated network environment where one VM can virtually attack another on the same host system.
Segregation and Isolation VMs promote security segregation and isolation, reducing the risk of infections transferring between different machines.
Sandboxing A suspicious program can be loaded into an isolated VM and executed (sandboxing). If the program is malware, only the virtual machine will be impacted, containing the threat.

Security Concerns for Virtual Environments

Virtualized environments face specific security concerns that must be addressed:

  • Hypervisor Security: Not all hypervisors have necessary security controls to keep out attackers. Existing security tools were often designed for single physical servers, making management complex in a virtual environment.
  • VM Escape: This occurs when a VM is able to "break out" from the contained environment and directly interact with the host OS. If successful, an attacker could potentially infect the host OS and spread malware to all other VMs running on it, necessitating virtual machine escape protection.
  • Virtual Machine Sprawl: The widespread proliferation of VMs without proper oversight or management. Forgotten VMs may lack the latest security updates, making them vulnerable when launched.
  • Virtual Machine Sprawl Avoidance: Combating sprawl involves performing regular audits, using good naming conventions, periodically cleaning up unnecessary VMs, and installing a virtual machine manager to provide a dashboard of VM status.

Host Hardening:
Another critical practice is using tools to harden the hosts. One such tool is Security-Enhanced Linux (SELinux), a security architecture for Linux systems. SELinux allows administrators greater control over system access by defining Access Control Lists (ACLs) for applications, processes, and files, and enforcing access via security policies. SELinux was originally developed by the National Security Agency (NSA).

Study Guide: Cloud and Virtualization Security

Quiz: Short-Answer Questions

Instructions: Click on a question to reveal the answer.

1. Describe the primary cost-saving factors associated with cloud computing.

The cost savings from cloud computing are attributed to elasticity and scalability, which allow resources to be expanded or reduced as needed; a pay-per-use model where organizations only pay for what they use; on-demand services that eliminate the need to purchase permanent hardware; and resiliency, where the cloud provider handles failures without user impact. This shifts costs from capital expenditures (CapEx) to operating expenses (OpEx).

2. What are the four main types of clouds, and how do they differ in terms of access?

The four main types of clouds are public, community, private, and hybrid. A public cloud offers services to all users over the Internet. A community cloud is open only to specific organizations with common concerns. A private cloud is created and maintained on a private network, offering the highest control. A hybrid cloud is a combination of public and private clouds.

3. Explain the concept of a "serverless infrastructure" and why the term is considered a misnomer.

A serverless infrastructure is a cloud architecture where the capacity planning, installation, setup, and management of servers are handled by the cloud provider and are invisible to the user. The term is a misnomer because physical servers are still performing the critical functions, but the end-user does not have to provision, deploy, or manage them.

4. What is the core function of a Cloud Access Security Broker (CASB), and where does it reside in a network architecture?

A CASB is a set of software tools or services that acts as a gatekeeper between an enterprise's on-premises infrastructure and the cloud provider's infrastructure. Its core function is to ensure that the enterprise's security policies, such as data encryption, extend to its data and applications in the cloud.

5. Define "live migration" in the context of virtualization and explain its primary benefits.

Live migration is a technology supported by virtualization that allows a virtual machine (VM) to be moved to a different physical computer with no impact on users. This provides uninterrupted server access during hardware maintenance and can be used for load balancing by moving high-demand VMs to servers with more resources.

6. How does a Software-Defined Network (SDN) differ from a traditional physical network?

An SDN virtualizes parts of the physical network by separating the control plane from the data plane. Unlike traditional networks where control logic is bundled with each switch, an SDN uses a centralized controller to manage complex functions like routing and security policies, making the network faster and easier to reconfigure.

7. What is virtual machine sprawl, and what are the security risks associated with it?

Virtual machine sprawl is the widespread proliferation of VMs without proper oversight or management. The associated security risk is that forgotten or dormant guest OSs may not receive the latest security updates, making them vulnerable to attack when they are eventually launched.

8. Explain the difference between a Type I and a Type II hypervisor.

A Type I hypervisor, also called a "native" or "bare metal" hypervisor, runs directly on the computer's hardware. A Type II hypervisor runs on top of the host operating system, much like a regular application.

9. Describe the purpose of "secrets management" within a microservices-based architecture.

Secrets management enables strong security for a microservices architecture by providing a central repository to manage, access, and audit "secrets" like API keys, passwords, and tokens. This allows the cloud infrastructure to remain flexible and scalable without hard-coding sensitive credentials into the software.

10. What is a Secure Access Service Edge (SASE), and what are some of the key technologies it incorporates?

A SASE is the convergence of several security services into a single, cloud-delivered service model that provides secure access based on user identity and real-time context. Key technologies that make up a SASE include Secure Web Gateway (SWG), Cloud Access Security Broker (CASB), zero-trust architecture (ZTA), and WAN technologies.
