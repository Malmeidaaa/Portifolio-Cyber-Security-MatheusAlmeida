Module 7: Identity and Access Management (IAM)

Controlling user validation and access to digital resources.

Identity and Access Management (IAM)

Identity and Access Management (IAM) is the foundational technology that provides control over user validation and the digital resources that may be accessed. The primary goal of IAM is to ensure that the right users access the right digital resources at the right time and for the right reasons. Essentially, the foundation of security is keeping unauthorized users out while letting legitimate users in.

In information security, IAM requires two main steps when a user accesses a device: first, providing some type of unique digital identity (the identity) and, second, providing proof that they are this unique user (identity proofing, also known as authentication).

1. Types of Authentication Credentials

Authentication is the process of ensuring that the person or system desiring access to resources is genuine and not an imposter. The elements that prove authenticity are called authentication credentials. There are seven elements that can be used as credentials. The most common elements found in IAM are "something you know," "something you have," and "something you are" (called factors), while the remaining four are called attributes.

Table 1: The Seven Elements of Authentication Credentials (Factors and Attributes)

Element Description Example Scenario
Something you know Knowledge that nobody else possesses Combination to unlock locker
Something you have Possession of an item that nobody else has Riker’s RFID card
Something you are Unique biological characteristic that cannot be changed Fingerprint reader to enter building
Somewhere you are Restricted location Restricted military base
Something you can do Perform an activity that cannot be exactly copied Paolo’s signature
Something you exhibit Genetically determined characteristic Peyton’s flaming red hair
Someone you know Validated by another person Li knows Peyton

1.1 Something You Know: Passwords

Passwords (a secret combination of letters, numbers, and/or characters) are the most common type of IAM authentication today. However, they provide weak protection and are constantly under attack.

How Passwords Work

When a user creates a password, the server converts the password into a scrambled set of characters, called a message digest (or hash), using a one-way hash algorithm. The original unscrambled password is not stored for security reasons. During authentication, the server creates a new digest of the entered password and compares it with the stored digest; if the two scrambled digests match, access is granted.

Password Weaknesses

The security of passwords relies on human memory, which, while immune to certain technological vulnerabilities, has limitations. The weakness stems from the burden of remembering long, complex, and unique passwords for numerous accounts (estimated at 207 per user on average). Furthermore, many policies mandate password expiration, forcing users to repeatedly memorize new ones.

Because of this burden, users take shortcuts that create vulnerable passwords, such as password reuse or the creation of predictable patterns like appending numbers (caitlin1) or substituting letters with numbers (passw0rd). Weak passwords are an easy attack vector, causing four out of every five enterprise data breaches.

Password Attacks

Attacks are classified into low-outcome and high-outcome categories:

Type of Attack Description Outcome
Online Brute Force Attack Continuous attempt of every possible combination (letters, numbers, characters) against a single account. Low Outcome. Rarely used because most accounts lock out after a limited number of incorrect attempts.
Password Spraying Attack Using one or a few common passwords (e.g., 123456) across several different user accounts. Low Outcome. Less likely to raise alarms or lock out user accounts.
Offline Brute Force Attack Uses a stolen password digest file. Password cracking software generates candidate digests of every possible character combination and matches them against the stolen file. High Outcome. Slowest yet most thorough method. GPUs are often used due to their superiority in repetitive tasks.
Dictionary Attack Creates digests of common dictionary words and phrases and compares them against the stolen digest file. High Outcome. Successful because users often create passwords from simple dictionary words.
Credential Stuffing Injection of stolen username and password credentials across multiple websites. High Outcome. Exploits password reuse by users.
Rule Attack Conducts a statistical analysis on stolen passwords to create a mask defining the format of the candidate password (e.g., ?u?l?l?l?l?d?d?d?d). High Outcome. Significantly reduces the time needed to crack a password.

1.2 Something You Have: Tokens and Security Keys

This second type of common credential relies on the approved user possessing a specific item.

Tokens and Multifactor Authentication (MFA)

A token is an object that can be hardware or software.

  1. Hardware Tokens (Windowed Tokens): Small devices displaying a dynamic value called a one-time password (OTP).
    • TOTP (Time-based): OTP that changes after a set period (e.g., every 30–60 seconds).
    • HOTP (HMAC-based): Event-driven OTP, changes when a specific event (like entering a PIN) occurs.
  2. Software Tokens: Typically generated through an app on a user’s smartphone.
    • Using more than one type of authentication credential (e.g., something you know + something you have) is called Multifactor Authentication (MFA).
    • A common software password supplement is an authentication app that sends a push notification to the smartphone, asking for approval or denial of the login.
    • Caution: Sending authentication codes via SMS text is not considered a secure option due to the risk of phishing and interception. "MFA fatigue" attacks exploit repeated notifications until the target accepts them to stop being bothered.
Security Keys and Smart Cards
  • Security Keys: A more secure option, these are dedicated dongles inserted into USB/Lightning ports or held near the device (NFC). They contain all the necessary cryptographic information for user authentication.
    • A feature is attestation, a key pair "burned" into the security key during manufacturing, specific to the device model, used to cryptographically prove the device's origin.
  • Smart Cards: Credit-card-sized plastic cards that hold information for the authentication process. The Personal Identity Verification (PIV) standard covers all U.S. government employees.

1.3 Something You Are: Biometrics

Biometrics relies on the unique features and characteristics of an individual. There are two main types: physiological and cognitive.

Physiological Biometrics

Uses a person’s unique physical characteristics for authentication.

Biometric Type Device/Characteristic Details
Retina Retinal Scanner (specialized) Maps unique retinal patterns by directing a beam of low-energy infrared light (IR) into the eye.
Fingerprint Static Scanner (takes a picture) or Dynamic Scanner (finger swiped across a slit) Most common. Locates the point where ridges end and split.
Vein Vein-scanning tablet (specialized) Identifies vein images in the palm or finger.
Gait Small sensors on a floor (specialized) Recognition of the manner of walking.
Voice Standard microphone Identifies users based on unique voice characteristics, including phonetic cadence.
Iris Standard webcam Identifies unique random patterns in the circular structure of the eye.
Facial Recognition Standard webcam Uses "landmarks" called nodal points (approx. 80) on the human face to create a numerical code (faceprint).

Biometric Disadvantages: Using biometrics has disadvantages, including the high cost of specialized hardware. Readers are not foolproof, resulting in false negatives (genuine users rejected) or false positives (imposters accepted). Biometric systems can also be "tricked" (e.g., fingerprints collected from water glasses). If digital biometric data is stolen, it cannot be reset like a password, potentially leaving the victim unable to use biometrics again.

Cognitive Biometrics

Relates to the perception, thought process, and understanding of the user. It is based on the user’s life experiences, making it easier for the user to remember and harder for an attacker to imitate. It is also called knowledge-based authentication.

  • Example: Microsoft’s Picture Password for Windows 10, where users select a picture with at least 10 "points of interest" and reproduce specific gestures (tap, line, circle) for authentication.

1.4 Something You Do: Behavioral Biometrics

Authenticates by normal actions the user performs.

  • Keystroke dynamics: Recognizes a user’s unique typing rhythm using two unique variables:
    1. Dwell time: The time it takes to press and release a key.
    2. Flight time: The time between keystrokes. This biometrics holds great potential because it requires no specialized hardware.

Authentication Best Practices

Authentication best practices involve securing passwords and using secure authentication technologies.

2.1 Protecting Password Digests

To secure stored password digests:

  • Salting: Consists of a random string (salt) added to the user's cleartext password before it is hashed. Salts make dictionary and brute force attacks much slower. They also ensure that identical passwords result in different digests. The salt should be random and unique for each user.
  • Peppering: The message digest is created normally but then encrypted with a symmetrical encryption key before being stored.
  • Key Stretching: A specialized password hash algorithm intentionally designed to be slower. This limits an attacker's speed in creating candidate digests. Using general-purpose hash algorithms, like MD5 and SHA, is not secure because they are too fast.
    • Common key stretching algorithms: bcrypt, PBKDF2, and Argon2.

2.2 Managing Passwords

Solutions focus on keeping individual user passwords safe:

  • Password Vaulting (Enterprise Level): Stores user password credentials in a highly protected database (vault) on the organization's network. Offers advanced capabilities like rotating and randomizing passwords.
  • User Password Manager: A software application or online website that stores multiple strong passwords in a single "vault" protected by one strong master password.
  • Hardware Password Key: Serves as a hardware-based password manager, more secure against malware than software vaults.

General Password Best Practices

  • Default passwords should be changed.
  • Password reuse should be prohibited.
  • Password expiration (the point in time when a password is no longer valid) should not be utilized as a best practice.
  • Password age (the period a password must be used before changing it) should be set to at least 1 day.
  • The most critical factor for a strong password is its length, not complexity. A longer password is always more secure.

2.3 Secure Authentication Technologies (SSO and Passwordless)

Single Sign-On (SSO) and Federation

  • Identity Management: Using a single authentication credential shared across multiple networks.
  • Federation (Federated Identity Management or FIM): Identity management when networks are owned by different organizations.
  • SSO: An application of federation using one authentication credential to access multiple accounts or applications, reducing the user's memorization burden.
Name Description Explanation
SAML (Security Assertion Markup Language) XML standard for secure web domains to exchange user authentication and authorization data. Allows login credentials to be stored with a single identity provider.
LDAP (Lightweight Directory Access Protocol) Makes it possible for almost any application to obtain directory information. The protocol enabling users to access network resources through a directory service.
OAuth (Open Authorization) Open-source federation framework. Supports the development of authorization protocols.
OpenID Open standard decentralized authentication protocol. Used in OAuth 2.0 as a standard means to obtain user identity.
Shibboleth Open-source software package for designing SSO. Uses federation standards to provide SSO and attribute exchange.

Passkeys

  • Passkeys are a passwordless alternative that stores authenticating information in hardware.
  • They combine multiple authentication factors (usually a device’s biometric scan or PIN) into a single package managed by the device’s OS, replacing traditional username/password/MFA.
  • Passkeys are "discoverable," meaning an enrolled device can automatically push a passkey through an encrypted tunnel to another device attempting to sign in.

Access Controls

Access control is the process of granting or denying approval to use specific resources once authenticated. In enterprise settings, it is essential to enforce least privilege, limiting access only to what is necessary for the user to complete their work.

An access control scheme is a predefined framework embedded in the software and hardware that administrators use for configuring the necessary level of control.

3.1 Access Control Schemes

Name Description Key Characteristics
Discretionary Access Control (DAC) Least restrictive scheme. Every object has an owner who has total control and discretion to grant permissions. Weaknesses: Relies on user decision to set security; permissions are inherited by programs executed by the subject. Example in Windows.
Mandatory Access Control (MAC) Most restrictive scheme. Assigns access controls strictly according to the custodian’s desires. End-users cannot set controls. Based on Labels (classification like confidential, secret) and Levels (hierarchy). Windows implementation is MIC (Mandatory Integrity Control).
Role-Based Access Control (RBAC) Based on a user’s job function within an organization. Assigns permissions to particular roles, and users are assigned to those roles. Considered a more "real-world" approach.
Rule-Based Access Control Dynamically assigns roles to subjects based on a set of rules defined by a custodian. Used for managing user access to one or more systems.
Attribute-Based Access Control (ABAC) Most flexible scheme. Uses policies that can combine attributes (object, subject, and environment attributes). Rules can be formatted using an If-Then-Else structure.

3.2 Access Control Lists (ACLs)

An Access Control List (ACL) is a set of permissions (authorizations) attached to an object. This list specifies who is allowed to access the object and what operations they can perform on it.

Although ACLs can be associated with any type of object, they are most often viewed in relation to providing filesystem permissions for protecting files managed by the Operating System (OS).

Limitations of ACLs: They can be inefficient, as the ACL for each resource must be checked upon every access. They are also difficult to manage in a large enterprise setting where many users require different access levels to many resources.

Study Guide for Module 7

Quiz: Short-Answer Questions

1. What are the three main categories of authentication factors?

Something you know (knowledge), something you have (possession), and something you are (inherence).

2. Why are passwords considered weak protection?

They rely on human memory, leading to reuse and simple patterns. They are also vulnerable to various attacks like brute force, dictionary, and phishing.

3. What is the difference between an Online Brute Force Attack and a Credential Stuffing attack?

Online Brute Force guesses passwords against a single account until locked out. Credential Stuffing uses stolen valid credentials (username/password pairs) from one breach to try and access accounts on other websites.

4. Explain the concept of "Salting" a password.

Salting involves adding a random string of characters to the password before it is hashed. This prevents identical passwords from producing the same hash and defends against rainbow table attacks.

5. What is TOTP and how does it differ from HOTP?

TOTP (Time-based One-Time Password) changes after a set period (e.g., 30 seconds). HOTP (HMAC-based One-Time Password) changes based on an event, such as pressing a button.

6. What is the False Acceptance Rate (FAR) in biometrics?

The rate at which a biometric system incorrectly accepts an unauthorized user (imposter) as a valid user.

7. Describe "Behavioral Biometrics" and give an example.

Authentication based on how a user performs an action. An example is keystroke dynamics, which measures dwell time (key press duration) and flight time (interval between keys).

8. What is Single Sign-On (SSO)?

A mechanism that allows a user to authenticate once and gain access to multiple independent software systems or applications without logging in again.

9. Which Access Control Scheme is considered the most restrictive and is based on labels/levels?

Mandatory Access Control (MAC).

10. What is the Principle of Least Privilege?

The security concept that a user should only be granted the minimum levels of access – or permissions – needed to perform their job functions.

Essay Questions

  2. Compare and contrast Discretionary Access Control (DAC) and Role-Based Access Control (RBAC). In which scenarios would DAC be appropriate? Why is RBAC generally preferred for medium-to-large enterprises?
  3. Evaluate the effectiveness of "passwordless" authentication technologies like Passkeys. How do they improve security over traditional passwords? What are the potential barriers to widespread adoption?
  4. Discuss the privacy and security implications of using biometric authentication. Consider issues such as the inability to reset compromised biometric data and the potential for "spoofing" sensors.
  5. Explain the importance of Identity and Access Management (IAM) as a foundational security layer. How does a robust IAM strategy support the concept of Zero Trust security?

Glossary of Key Terms

Term Definition
Authentication The process of verifying the identity of a user, device, or system.
Authorization The process of granting or denying specific permissions to an authenticated user.
Biometrics Authentication methods based on unique physical (physiological) or behavioral characteristics of an individual.
Brute Force Attack A trial-and-error method used to obtain information such as a user password or PIN.
Credential Stuffing A cyberattack where stolen account credentials are used to gain unauthorized access to user accounts through large-scale automated login requests.
DAC (Discretionary Access Control) An access control model where the owner of the object controls who has access to it.
Federation A method of linking a user's identity across multiple distinct security domains.
IAM (Identity and Access Management) A framework of policies and technologies for ensuring that the proper people in an enterprise have the appropriate access to technology resources.
MFA (Multi-Factor Authentication) An authentication method that requires the user to provide two or more verification factors to gain access.
RBAC (Role-Based Access Control) An access control method that restricts network access based on the roles of individual users within an organization.
Salt Random data that is used as an additional input to a one-way function that hashes data, a password or passphrase.
SSO (Single Sign-On) An authentication scheme that allows a user to log in with a single ID and password to any of several related, yet independent, software systems.
TOTP (Time-based One-Time Password) A temporary passcode generated by an algorithm that uses the current time of day as one of its authentication factors.
